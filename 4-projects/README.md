# 4. Projects

One subfolder per project. Projects are time-bounded; products are ongoing.

## Adding a project

Copy `_template/` into a numbered folder — `1-project-name/`. Start with `1-scope.md` — it defines everything else.

## What each project folder contains

```
1-[project-name]/
  README.md            what this project is and who leads it
  1-scope.md           objectives, deliverables, out-of-scope
  2-plan.md            timeline, milestones, risks
  3-decisions/         ADRs — decisions made during this project
  4-meetings/          meeting notes
  5-retrospective.md   learnings (filled when the project closes)
```

## For agents

Load the relevant project folder before any project-specific task. Scope and plan tell you what's committed, what's not, and who owns what.

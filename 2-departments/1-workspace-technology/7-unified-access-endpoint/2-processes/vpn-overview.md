# 2.7.2. Cisco AnyConnect VPN — Generic Information

## Important Notes

- Make sure your company is on-boarded to this VPN solution. If not, please contact Service Management to have this set up. A list of companies that are already on-boarded can be found in the internal resources.
- In order to log in to VPN, your **internal AD account** will have to be added to the proper company VPN group. This is done per request via **Visma MimPortal** and manager approval is required.
- A HowTo in relation to MimPortal VPN group request can be found in the internal resources.
- If you do not have 2FA (a.k.a. Google Authenticator) set up for your account, you will get prompted to enter an SMS code sent to the phone number registered on your AD account.
- The following HowTos are valid for both variants — 2FA or SMS.
- If you want to have 2FA set up on your user, please open a ticket at [it-servicedesk@visma.com](mailto:it-servicedesk@visma.com).
- **Login is not possible if there is no token assigned to your user (SMS or Google Auth).**

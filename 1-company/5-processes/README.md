# Company-Level Processes

Cross-company processes that apply across all departments and are not owned by any single team.

Department-specific processes belong in `2-departments/<name>/2-processes/` instead.

| File | Content |
|------|---------|
| `process-template.md` | Template for all company-level process docs |
| *(add processes here)* | e.g. `onboarding.md`, `hiring.md`, `procurement.md` |

**Common examples:**
- Employee onboarding (first day, access provisioning, equipment)
- Hiring process (job posting → interview → offer → start)
- Procurement / vendor approval
- Expense and reimbursement
- Security incident response
- Knowledge base governance (how the KB itself is maintained)
